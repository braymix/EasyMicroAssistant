🤖 Knowledge Assistant

Assistente AI locale — Quick Start Guide

⏱ 20–30 min al primo avvio
Requisiti: 🪟 Win Windows 10/11  ·  🍎 Mac macOS 12+ (Intel o Apple Silicon)  ·  8 GB RAM min  ·  15 GB disco libero  ·  Docker Desktop

Avvio in 5 Step

STEP 1

🐳 Installa Docker Desktop

  1. Vai su docker.com/products/docker-desktop
  2. 🪟 Installa il .exe — spunta "Use WSL 2" ✅ — riavvia il PC
  3. 🍎 Trascina Docker in Applicazioni, aprilo, accetta i permessi
  4. Aspetta che la 🐳 (system tray / menu bar) diventi verde
STEP 2

📁 Scarica il Progetto

  1. Estrai lo ZIP ricevuto in una cartella
  2. 🪟 es. C:\Progetti\knowledge-assistant
  3. 🍎 es. ~/Progetti/knowledge-assistant
Oppure via Git:
git clone -b main https://github.com/braymix/easymicroassistant.git
STEP 3

⚙️ Avvia il Setup Automatico

🪟 Windows
  • Apri la cartella scripts\
  • Doppio click su setup-windows.bat
🍎 macOS
  • Apri il Terminale nella cartella del progetto
  • Esegui:
chmod +x scripts/*.sh
./scripts/setup-mac.sh
Quando chiede la modalità → digita 1 e premi Invio. Aspetta 10–15 min.
✅ "SETUP COMPLETATO CON SUCCESSO!"
STEP 4

🌐 Apri l'Interfaccia

  1. Apri il browser
  2. Vai su http://localhost:3000
  3. Clicca "Sign Up" e crea il tuo account
⚠️ Il primo account diventa admin.
I dati restano solo sul tuo PC.
STEP 5

🧠 Scarica un Modello AI

Se lo script non lo ha già fatto:

  1. Clicca "Select a model" in alto a sinistra
  2. Scrivi llama3.2:3b
  3. Clicca "Pull" e aspetta (~2 GB)

Usare la Knowledge Base

📚

1 — Crea la KB

Workspace → Knowledge → + → dai un nome (es. "Byblos") → Create

📄

2 — Carica Documenti

Clicca + dentro la KB → Upload Files → scegli PDF, TXT, MD, DOCX

💬

3 — Usa in Chat

Scrivi # nel campo messaggio → seleziona la KB → fai la domanda

Comandi Rapidi

Azione 🪟 Windows 🍎 macOS
Avviare il sistema scripts\start-full.bat ./scripts/start-full.sh
Fermare il sistema scripts\stop.bat ./scripts/stop.sh
Verificare lo stato scripts\status.bat ./scripts/status.sh
Scaricare un modello scripts\pull-model.bat ./scripts/pull-model.sh
Fare un backup scripts\backup.bat ./scripts/backup.sh
Diagnostica errori scripts\test-setup.bat ./scripts/test-setup.sh

Problemi Frequenti

ProblemaSoluzione
Pagina non si apre Avvia Docker Desktop, aspetta balena verde 🐳, poi lancia start-full
Nessun modello disponibile Esegui pull-model → scrivi llama3.2:3b → Invio
Risposte molto lente Usa modello più piccolo: llama3.2:3b invece di 8b
KB non trova informazioni Controlla spunta ✅ sul file in Workspace → Knowledge
🍎 "Permesso negato" sullo script chmod +x scripts/*.sh nel Terminale
